# ADR 002: ساختار پنج‌گانه taxonomy برای زمین X

## وضعیت
پذیرفته‌شده

---

## زمینه

زمین X قرار است مجموعه‌ای از قابلیت‌های قابل بازاستفاده، الگوهای کاربردی، ساختارهای آماده و اپلیکیشن‌های کاربردی کوچک باشد.

برای اینکه مرز اجزا روشن بماند، ساختار ریپو، مستندات و نقشه ماژول‌ها باید بر پایه یک taxonomy پایدار تعریف شوند.

اگر این taxonomy روشن نباشد:
- مرز ماژول‌ها مبهم می‌شود
- جایگذاری capabilityها سلیقه‌ای می‌شود
- مستندسازی ناهماهنگ می‌شود
- backlog و roadmap مبنای پایدار نخواهند داشت

---

## تصمیم

زمین X بر اساس پنج دسته اصلی سازمان‌دهی می‌شود:

- BuildingBlocks
- ApplicationPatterns
- Integrations
- Foundations
- Applications

---

## دلایل تصمیم

این تصمیم برای این گرفته می‌شود که:
- مرز انواع دارایی‌های محصول روشن شود
- نقشه ماژول‌ها ساختار پایدار پیدا کند
- تصمیم‌گیری درباره جایگذاری ماژول‌ها ساده‌تر شود
- مستندات و backlog بر یک مدل مشترک سوار شوند

---

## پیامدها

### پیامدهای مثبت
- ساختار پروژه و docs روشن‌تر می‌شود
- گفت‌وگوی معماری بر پایه یک مدل مشترک انجام می‌شود
- توسعه ماژول‌های جدید قابل هدایت‌تر می‌شود

### هزینه‌ها
- بعضی دارایی‌های فعلی باید بازطبقه‌بندی شوند
- نیاز به تعریف دقیق مرز هر دسته وجود دارد
