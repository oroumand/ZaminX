# پشتیبانی از چند فراهم‌کننده داده با ترتیب و override در Translation

## وضعیت

پذیرفته‌شده

---

## زمینه

در capability ترجمه، داده‌ها می‌توانند از sourceهای مختلف تأمین شوند، مانند:

- دیتابیس
- فایل‌ها
- cache
- یا سایر sourceها

در بسیاری از سناریوهای واقعی، نیاز است:

- ترجمه‌ها از چند source بارگذاری شوند
- برخی sourceها نقش پایه داشته باشند
- برخی sourceها مقادیر قبلی را override کنند

اگر capability فقط از یک فراهم‌کننده داده پشتیبانی کند:

- ترکیب چند source امکان‌پذیر نیست
- مدیریت overrideها پیچیده می‌شود
- انعطاف‌پذیری سیستم کاهش می‌یابد

برای پشتیبانی از این سناریوها، لازم است capability بتواند چند فراهم‌کننده داده را به‌صورت هم‌زمان مدیریت کند.

---

## تصمیم

capability Translation (Parrot) از چند فراهم‌کننده داده به‌صورت هم‌زمان پشتیبانی می‌کند.

در این مدل:

- چند فراهم‌کننده داده می‌توانند در زمان registration معرفی شوند
- ترتیب ثبت آن‌ها حفظ می‌شود
- داده‌ها به ترتیب از فراهم‌کننده‌ها بارگذاری می‌شوند
- در صورت وجود کلید تکراری، مقدار فراهم‌کننده‌ای که دیرتر ثبت شده است جایگزین مقدار قبلی می‌شود

## پیامدها

### مثبت

- امکان ترکیب چند source ترجمه فراهم می‌شود
- override کردن داده‌ها بدون تغییر source اصلی ممکن می‌شود
- انعطاف‌پذیری capability افزایش می‌یابد
- هم‌راستایی با الگوی configuration در .NET

### منفی

- ترتیب registration اهمیت پیدا می‌کند
- نیاز به پیاده‌سازی منطق merge در داخل capability
- افزایش پیچیدگی در مدیریت داده‌ها
