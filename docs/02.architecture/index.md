# نمای کلی معماری

## هدف این سند
این سند معماری کلان زمین X را در سطح بالا تعریف می‌کند.

نقش این سند:
- تعریف دسته‌های اصلی معماری
- روشن کردن مرز هر دسته
- مشخص کردن جهت کلی وابستگی‌ها
- ایجاد ورودی برای بخش ماژول‌ها، تصمیم‌های معماری و backlog

این سند وارد طراحی تفصیلی ماژول‌ها یا جزئیات پیاده‌سازی نمی‌شود.

---

## ساختار کلان

زمین X بر اساس پنج دسته اصلی سازمان‌دهی می‌شود:

### 1. BuildingBlocks
اجزای مستقل، خودبسنده و قابل بازاستفاده که می‌توانند به‌تنهایی استفاده شوند.

### 2. ApplicationPatterns
الگوهای پرکاربرد سطح اپلیکیشن که معمولاً از چند BuildingBlock استفاده می‌کنند.

### 3. Integrations
اجزایی که مسئول اتصال به سرویس‌ها، سیستم‌ها و مرزهای بیرونی هستند.

### 4. Foundations
ساختارهای آماده، قالب‌ها و سازوکارهای ایجاد پروژه.

### 5. Applications
اپلیکیشن‌های کوچک، قابل اجرا و ارزشمند که با استفاده از زمین X ساخته می‌شوند.

---

## تعریف دسته‌ها

### BuildingBlocks
BuildingBlocks بر اساس استقلال مصرف، خودبسندگی و قابل بازاستفاده بودن تعریف می‌شوند.

در این دسته:
- وابستگی به تکنولوژی مانع قرارگیری نیست
- وجود انتزاع الزامی نیست
- اگر بستر .NET انتزاع مناسبی داشته باشد، از همان استفاده می‌شود
- setup، registration و provider نیز در صورت مستقل بودن می‌توانند در همین دسته قرار بگیرند

همچنین بعضی اجزای pattern-like مانند `Entity`، `AggregateRoot`، `ValueObject`، `DomainEvent` و `Mediator` نیز در صورتی که به‌صورت مستقل و قابل بازاستفاده ارائه شوند، در همین دسته قرار می‌گیرند.

---

### ApplicationPatterns
ApplicationPatterns الگوهای پرکاربرد سطح اپلیکیشن هستند که از چند جزء موجود استفاده می‌کنند تا یک رفتار تکرارشونده را به‌صورت استاندارد ارائه دهند.

نمونه‌های مورد انتظار:
- `Outbox`
- `Inbox`
- `Polling Publisher`
- `Event Handling`
- `CQRS`

---

### Integrations
Integrations زمانی معنا پیدا می‌کنند که مسئله اصلی، اتصال به یک boundary بیرونی باشد.

نمونه‌های مورد انتظار:
- یکپارچه‌سازی با message brokerها
- یکپارچه‌سازی با providerهای هوش مصنوعی
- یکپارچه‌سازی با موتورهای جست‌وجو
- یکپارچه‌سازی با file storage یا object storage
- یکپارچه‌سازی با سرویس‌های بیرونی

---

### Foundations
Foundations شامل artifact قابل اجرا نیستند، بلکه ساختارهای آماده و opinionated برای شروع پروژه‌ها هستند.

جهت‌گیری فعلی این دسته بر پایه این موارد است:
- `MonolithStructure`
- `ModularMonolith`

همچنین `MicroserviceModule` فعلاً به‌عنوان گزینه باز برای آینده در نظر گرفته می‌شود.

---

### Applications
Applications اپلیکیشن‌های کوچک، قابل اجرا و ارزشمند هستند که برای استفاده مستقیم، شروع سریع توسعه یا نمایش استفاده واقعی از اجزای زمین X ارائه می‌شوند.

نمونه‌های مورد انتظار:
- اپ مدیریت ترجمه
- اپ RAG سبک
- اپ مشاهده و تحلیل تاریخچه تغییرات و ممیزی

---

## روابط بین دسته‌ها

جهت کلی وابستگی‌ها در زمین X به این صورت است:

- ApplicationPatterns می‌توانند به BuildingBlocks وابسته باشند
- Integrations می‌توانند برای اتصال به بیرون از BuildingBlocks و در صورت نیاز از ApplicationPatterns استفاده کنند
- Foundations می‌توانند از BuildingBlocks، ApplicationPatterns و Integrations استفاده کنند
- Applications می‌توانند از همه دسته‌های پایین‌تر استفاده کنند

وابستگی معکوس مجاز نیست.
